## 一、已修复的严重问题

### P0-1：英语 Boss 阅读理解题数据结构错乱（已修复 ✅）
- **影响范围**：en-day6 到 en-day40，共 35 天的 Boss 第 1 题（index 28）
- **问题描述**：`options` 字段被错误地设为字符串（问题文本），`answer` 字段被设为数组（选项列表），导致游戏引擎调用 `.map()` 时会崩溃
- **根因**：生成脚本 `BR()` 函数输出 JSON 时字段顺序错位
- **修复方式**：将 passage→`passage`，question→`question`，options array→`options`，answer index=0

### P0-2：`_finishAndSave` 正确率超过 100%（已修复 ✅）
- **影响范围**：所有做完错题重做后保存的记录
- **问题描述**：`correctCount` 同时累加了 `allAnswers.filter(isCorrect)` 和 `reviewCorrectCount`，导致重做对的题被计算两次。模拟测试显示正确率可达 167%
- **根因**：`_continueReview` 已经把 `origRec.isCorrect` 设为 true，`allAnswers.filter(isCorrect)` 已经包含重做对的题，再 `+ reviewCorrectCount` 就是重复计算
- **修复方式**：移除 `+ (state.reviewCorrectCount || 0)`，`totalQuestions` 和 `correctCount` 都只基于 `allAnswers`

### P1-1：数学计算题选项完全重复（已修复 ✅）
- **影响范围**：8 道分数计算题（math-day18/19/30/31/32）
- **问题描述**：如 `["1/6","1/6","1/2","2/5"]`，两个选项完全相同
- **修复方式**：替换重复项为不同干扰项

### P1-2：英语 Boss 题选项不足 4 个（已修复 ✅）
- **影响范围**：en-day2（2 个选项）、en-day3（3 个）、en-day5（2 题 × 3 个）
- **修复方式**：补充干扰选项至 4 个

### P1-3：数学计算题等价分数干扰项（已修复 19/38 ✅）
- **影响范围**：math-day17-20, 30-32, 37-40 的计算题
- **问题描述**：如 `["7/6","14/27","21/18","7/6"]`，其中 21/18 = 7/6，学生选了等价分数会被判错
- **修复方式**：自动替换等价分数为非等值干扰项（已修复 19 道，剩余 19 道需手动检查）

## 二、待处理问题

### P2-1：剩余 19 道数学计算题等价分数干扰项
- **位置**：math-day16/17/18/19/20/30/38/40
- **建议**：逐题检查，将等价分数替换为不同值的干扰项
- **注意**：Day 13-14（约分通分）和 Day 24-26（倒数/约分）中的等价分数是**教学需要**，不应修改

### P2-2：计时器到 0 不自动提交
- **位置**：`game-engine.js` `_startTimer()` 第 418 行
- **现象**：计时器归零后仅清除定时器，题目仍留在屏幕上，学生可无限时间作答
- **影响**：计时器形同虚设，Boss 关"限时挑战"无实际意义
- **建议**：归零后自动判错或自动跳过，至少在 Boss 关执行

### P2-3：`timeSpent` 硬编码为 5 秒
- **位置**：`game-engine.js` 第 477 行 `timeSpent: 5`
- **现象**：所有答案记录的耗时都是 5 秒，不反映真实答题时间
- **影响**：学习看板无法分析学生答题速度
- **建议**：记录每题开始时间，结束时计算实际耗时

### P2-4：ffmpeg 路径硬编码
- **位置**：`server.js` 第 53 行 `const FFMPEG = '/opt/homebrew/bin/ffmpeg'`
- **影响**：在 Linux 服务器或 Windows 上部署时找不到 ffmpeg
- **建议**：改为 `const FFMPEG = process.env.FFMPEG_PATH || 'ffmpeg'`，依赖系统 PATH

### P2-5：39 道概念题的等价分数选项（设计如此，需确认）
- **位置**：math-day13/14/24/25/26
- **说明**：这些是约分、通分、倒数等概念题，选项中有等价分数是教学需要。但需确认题目表述是否清楚（如"选出最简分数"vs"选出相等的分数"）

## 三、代码质量与架构

### 优点 👍
1. **清晰的 SPA 架构**：hash 路由 + 页面切换，模块化良好
2. **数据持久化完善**：localStorage + JSON 导出/导入 + CSV 导出
3. **游戏引擎设计合理**：5 关卡 + 权重分配 + 选项打乱 + 错题重做
4. **多用户支持**：用户管理 + 排名 + 打卡 + 星星系统
5. **口语评测三层凭证链**：环境变量 → .env → 全局凭证文件
6. **前端零 console.log**：生产代码干净
7. **Cloudflare Worker 备选方案**：支持无服务器部署
8. **题型分布均匀**：英语 10词+8语法+5听力+5跟读+3Boss，数学 12计算+8概念+3Boss
